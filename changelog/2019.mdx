# 2019 Data Release Log

<Update label="December 2019" description="Holiday Maintenance Release" tags={["Data Update"]}>

<Update label="November 2019" description="Merging Logic Overhaul" tags={["Schema Change", "Completed"]}>
### Merging Logic Overhauled

Merging logic now includes field weighting — certain fields override others if present, reducing unnecessary or invalid merging between sources.

### Multi-Month Schema Updates Completed

- **Completed:** `crawlResultFiles` → `crawlIDs` transition
- **Completed:** `geoLocation` for Business and Property data
- **Completed:** `dateSeen` single-value transition
- **Completed:** UPC/EAN/`gtins` field rollout
- **Completed:** Bad seller ranking data removal from categories and features

### New Sources

- 2 Business data sources
- 1 Property data source

### Fixes

- Fixed ASIN records to use lowercase key values
- Updated 4 Property, 2 Product, 1 Business sources
</Update>

<Update label="October 2019" description="Database Tooling Overhaul Begins" tags={["Schema Change", "New Feature"]}>
### Database Tooling Overhaul

Database change tooling overhaul began — future schema changes and fixes will roll out faster.

### Schema Changes

- Added `naics` data to People data
- `mostRecentStatus` now source-dependent — individual domains provide status instead of deriving it
- `gtins` check digit verification added to validate global product identifiers
- In-progress: `crawlResultFiles` → `crawlIDs`, `geoLocation`, `dateSeen`, UPC/EAN/`gtins`

### New Sources

- 4 Business data sources

### Fixes

- `mostRecentStatusDate` now collected from sources instead of derived from crawl timestamps
- Address normalization to USPS values ongoing
- Updated 6 Property, 2 Product, 1 Business sources
</Update>

<Update label="September 2019" description="Property Schema Updates" tags={["Schema Change", "New Feature"]}>
### New Property Fields

- `mostRecentStatusDate` — Date a status took place

### Schema Updates

- `imageURLs` now retains only the most recent links per domain instead of compiling over time
- In-progress: `crawlResultFiles` → `crawlIDs` (Product remaining), `geoLocation`, `dateSeen`, UPC/EAN/`gtins`

### New Sources

- 2 Business data sources
- 1 Product data source
- 1 Property data source

### Fixes

- Updated 5 Business, 3 Property, 1 Product sources
</Update>

<Update label="August 2019" description="Schema Revisions" tags={["Schema Change", "New Feature"]}>
### New Product Fields

Added `upca`, `upce`, `ean13`, `ean8`, and `gtins` for better product search. Original `upc` and `ean` fields supported until deprecated.

### Schema Updates

- `dateSeen` in nested JSON objects now set to single latest-date value (future rename to `dateUpdated`)
- Added `geoLocation` field to Business and Property data combining latitude and longitude for future distance queries
- Continuing transition from `crawlResultFiles` to `crawlIDs`

### New Sources

- 3 Business data sources
- 1 Product data source

### Fixes

- Bad seller ranking data polluting categories and features — removal in-progress
- Removed "Details about" prefix appearing in Product name field
- Updated 1 Business, 4 Property, 2 Product sources
</Update>

<Update label="July 2019" description="Initial Release Log" tags={["Announcement"]}>
### First Data Release Log

Initial release log with the goal of tracking changes to the data — record count updates, new sources, schema changes, and fixes.

### Schema Changes

- Added `linkedinURL` to People data for LinkedIn profile URLs
- Added `primaryImageURLs` to Product data to track the main image per domain
- Began transition from `crawlResultFiles` to `crawlIDs`

### New Sources

- 2 Business data sources
- 1 Property data source
- 2 Product data sources

### Fixes

- Property `dateUpdated` now only changes when new source data is received, not on any modification
