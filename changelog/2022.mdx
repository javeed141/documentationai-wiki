# 2022 Data Release Log

<Update label="2022-12" description="Commercial Property & Brand Normalization" tags={["schema changes", "new sources", "fixes"]}>
  New commercial property sources for Canadian and US markets. Product brand normalization. Enrichment API improvements. End-of-year project assessments.

  **Record Counts:** Business 126.8M · People 3M · Product 489.6M · Property 171.2M

  **Source Updates:**
  - Added CBRE.ca and CBRE.com for commercial properties — over 20,000 new records
  - allmenus.com added as Business source for delivery and fast-service restaurants
  - Added Amazon.co.uk and Amazon.ca

  **Schema Changes:**
  - Added `canonicalBrand` top-level Product field for normalized brand names from scraped data
  - `brand` field now stores the raw scraped text of the product brand name

  **API Updates:**
  - Enrichment API improved across all data types — 4.5x speed increase for enrichment calls
  - New logic to improve product brand matching upon import and validation

  **Fixes:**
  - Cleaned ~100,000 property records with incorrect normalization causing duplicate/confusion in searches
  - Fixed Zoopla.co.uk crawler logic where HTML changes caused scraping failures
</Update>

  **Record Counts:** Business 126.1M · People 3M · Product 478.6M · Property 171.2M

  **Source Updates:**
  - Updated Redfin.com and Compass.com crawl patterns for increased broker data coverage
  - allmenus.com added as Business source for delivery and fast-service restaurants
  - Added Amazon.co.uk and Amazon.ca

  **Schema Changes:**
  - `licenseNumber` added to Property data for realtor/agent license numbers (when obtainable from source)

  **Portal & API Updates:**
  - Country and province dropdown filters added to Portal Filter Builder
  - Data enrichment feedback requested from users to improve enrichment capabilities

  **Fixes:**
  - Fixed views not showing for Bronze users
  - Patched property records street abbreviation issue (identified Oct 18, patched Oct 21) — optimized normalization and merge logic for records from data scraper
</Update>

</Update>
