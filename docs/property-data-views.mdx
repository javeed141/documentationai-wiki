## What are views?

Views are predefined or custom field selections that control:

- Which fields are included in the response
- How nested fields are flattened
- The structure of the output data

## Benefits of using views

- **Reduced data transfer** - Only receive the fields you need
- **Faster processing** - Smaller responses are quicker to parse
- **Cleaner data** - Focus on relevant information
- **Cost efficiency** - Optimized for your specific use case

## Using views in API requests

Add the `view` parameter to your request:

```json
{
  "query": "country:US",
  "num_records": 10,
  "view": "property_basic"
}
```

Or specify fields directly:

```json
{
  "query": "country:US",
  "num_records": 10,
  "view": "address,city,province,numBedroom,numBathroom,prices"
}
```

<Callout kind="tip">
  Start with a built-in view and customize from there based on your needs.
</Callout>
