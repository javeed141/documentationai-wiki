## Address normalization

Address data is standardized to follow a consistent format:

| Original | Normalized |
|----------|------------|
| `123 Main Street` | `123 Main St` |
| `456 North Oak Avenue` | `456 N Oak Ave` |
| `789 East 5th Street, Apt 2B` | `789 E 5th St Apt 2B` |

## Searching addresses

When searching by address, you can use either the full or abbreviated form:

```
address:"123 Main St"
address:"123 Main Street"
```

Both queries will match the same records.

<Callout kind="tip">
  For best results, search using the normalized abbreviated format.
</Callout>

## Address components

Addresses are broken into separate fields for flexible querying:

| Field | Example |
|-------|---------|
| `address` | `123 Main St` |
| `city` | `Austin` |
| `province` | `TX` |
| `postalCode` | `78701` |
| `country` | `US` |

## Combining address fields

Search for properties in a specific location:

```
address:"123 Main St" AND city:"Austin" AND province:"TX"
```

Or search by area:

```
city:"Austin" AND province:"TX" AND postalCode:78701
```
