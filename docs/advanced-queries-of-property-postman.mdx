This will walk you through querying the Property Database using Postman!

## 1. Select the Property Database

<Image src="https://files.readme.io/3d5a5f8-1.jpg" width="2560" height="1600" alt="Click on &#x22;Search Property Data&#x22; to get started." />

## 2. Setting the Stage

<Image src="https://files.readme.io/1ffc2eb-2.jpg" width="2560" height="1600" alt="Click on the &#x22;Body&#x22; tab and make sure you have &#x22;Raw&#x22; selected and that your text is set to &#x22;JSON(application/JSON)&#x22;." />

## 3. Running a Query

<Image src="https://files.readme.io/cf2aaa5-3.jpg" width="2560" height="1600" alt="The &#x22;Body&#x22; window will have 2 fields in it already. Hitting &#x22;Send&#x22; will query the Property Database for all records with a key." />

<Callout kind="info" collapsed="false">
  **The "Wild Card"**

  Using **\*** in your query will search for any possible value that could fill that spot. In the above case, it is essentially saying "Find all records that have a key value set to anything"
</Callout>

You can query the database using most of the fields in the [Property Data Schema](https://docs.datafiniti.co/docs/property-data-schema).

## 4. Narrowing The Search

<Image src="https://files.readme.io/58eaeeb-4.jpg" width="2560" height="1600" alt="You can search with different fields together inside the &#x22; &#x22; of your query string using AND or OR. The ( ) tells the API to process both values--trulia.com and movoto.com--as the sourceURLs of the record. Because they are connected with OR, we will see records that have one or the other. Had we used AND, only records with both sourceURLs would be shown." />

## 5. Using "" Inside the Query String

<Image src="https://files.readme.io/52de5e9-5.jpg" width="2560" height="1600" alt="To refine your query more, you may want to search for a specific word and not allow for any variations on it. To do that, you put it inside quotes. However, because you are already inside the main quotes of your string, you need to escape the new quotes by adding \ in front of each &#x22; inside the main quotation marks. See the treatment of \&#x22;For Sale\&#x22; value in the above example for clarification." />

## 6. Additional Specifications

<Image src="https://files.readme.io/da92c3f-6.jpg" width="2560" height="1600" alt="In addition to the &#x22;query&#x22; and &#x22;num_records&#x22;, there are other values you can specify in the body text. You can set &#x22;download&#x22; to either be true or false, &#x22;format&#x22; to either be &#x22;CSV&#x22; or &#x22;JSON&#x22;, and you can call any &#x22;view&#x22; you have saved to your account." />

<Callout kind="info" collapsed="false">
  **View**

  Setting your `"view"` not only changes the layout of your result file, but also changes how you see the records in Postman.
</Callout>

## 7. Custom Views on the Fly

<Image src="https://files.readme.io/496ee3c-7.jpg" width="2560" height="1600" alt="If you'd like to create a custom view that you don't want to save, you can customize it inside the &#x22;view&#x22; field before running your query!" />

<Callout kind="info" collapsed="false">
  **Custom View Formatting**

  When creating the `"view"` object, put the entire thing inside of `[ ]`, with each field being called inside of `{ }`. If you are listing multiple fields, put a comma after each one except for the last one. See the screenshot above for an example.
</Callout>
