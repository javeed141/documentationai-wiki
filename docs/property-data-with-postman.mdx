---
title: "Property Data with Postman"
description: "Learn how to use Datafiniti's Property Data API with Postman to search, filter, and download property records."
---

For this guide, we're going to assume you're interested in using Datafiniti's property data to do some marketing analysis on homes in the US. Let's say you're a data scientist that's been tasked with the following:

1. Collect data on homes
2. Sort the data by state
3. Find which states have the most expensive homes

**Your environment and data needs:**

- You're working with Postman
- You want to work with CSV or JSON data

<Steps>
  <Step title="Download Postman" icon="download" title-type="h3">
    We recommend using Postman when accessing the Datafiniti API. It's a great client for using any sort of RESTful API and works particularly well with Datafiniti.

    [Download Postman](https://www.postman.com/downloads/)
  </Step>

  <Step title="Get your API token" icon="key" title-type="h3">
    The API token lets you authenticate with Datafiniti API and tells it who you are, what you have access to, and so on. Without it, you can't use the API.

    To get your API token:
    1. Go to the [Datafiniti Web Portal](https://portal.datafiniti.co/settings/api)
    2. Login and click your settings in the left navigation bar
    3. Copy your API token (a long string of letters and numbers)

    <Callout kind="info">
      For the rest of this document, we'll use `AAAXXXXXXXXXXXX` as a substitute example for your actual API token.
    </Callout>

    <Callout kind="alert">
      For security reasons, your API token will be automatically changed whenever you change your password.
    </Callout>
  </Step>

  <Step title="Run your first search" icon="search" title-type="h3">
    The first thing we'll do is a test search that will give us a sense for what sort of data might be available.

    With Postman open, click on the Datafiniti API V4 collection. Then navigate to **Secured Endpoints > Search > Search Property Data**. Click on "Body" in the main screen.

    Enter the following in the text area:

    ```json
    {
      "query": "country:US",
      "num_records": 1
    }
    ```

    Click the blue "Send" button. You should get a response similar to this:

    ```json
    {
      "num_found": 7983205,
      "total_cost": 1,
      "records": [
        {
          "address": "711 Kent Ave",
          "brokers": [
            {
              "agent": "Raj Singh",
              "company": "YOUR REALTY INC.",
              "dateSeen": ["2016-06-06T18:09:28Z"]
            }
          ],
          "city": "Catonsville",
          "country": "US",
          "dateAdded": "2016-06-06T18:09:28Z",
          "latitude": "39.284462",
          "longitude": "-76.734069",
          "numBathroom": 2,
          "numBedroom": 4,
          "postalCode": "21228",
          "propertyType": "Single Family Dwelling",
          "province": "MD",
          "id": "AV9WzHyO_RWkykBuv11F"
        }
      ]
    }
    ```

    ### Request parameters

    | Parameter | Description |
    |-----------|-------------|
    | `query` | Tells the API what you want to search. In this case, searching by `country` returns any property in the US. |
    | `num_records` | Tells the API how many records to return. Here we just want 1 matching record. |
    | `format` | Tells the API what format to output (CSV or JSON). |

    ### Response fields

    | Field | Description |
    |-------|-------------|
    | `num_found` | Total number of available records matching your query. |
    | `total_cost` | Number of credits this request cost. Property records cost 1 credit per record. |
    | `records` | The matching records. If no matches, this field will be empty. |
  </Step>

  <Step title="Refine your search" icon="filter" title-type="h3">
    The original query didn't specify we only wanted homes for sale. Let's narrow our search:

    ```json
    {
      "query": "country:US AND propertyType:\"Single Family Dwelling\"",
      "format": "CSV",
      "num_records": 10
    }
    ```

    This API call:
    - Adds `AND propertyType:"Single Family Dwelling"` to narrow results
    - Changes `num_records` to `10` to see more samples
    - Adds `format` to specify CSV output

    <Callout kind="tip">
      Datafiniti lets you construct very refined boolean queries using AND, OR operations, negation, and more. Place search terms in quotation marks for exact matches.
    </Callout>
  </Step>

  <Step title="Initiate a download" icon="download" title-type="h3">
    Once you like what you see from the sample matches, it's time to download a larger data set:

    ```json
    {
      "query": "country:US AND propertyType:\"Single Family Dwelling\"",
      "format": "CSV",
      "num_records": 50,
      "download": true
    }
    ```

    Changes made:
    - `num_records` set to `50` to download 50 matching records
    - `download` set to `true` to issue a download request

    <Callout kind="alert">
      If `num_records` is not specified, ALL records matching the query will be downloaded.
    </Callout>

    You'll receive a response like:

    ```json
    {
      "id": 7,
      "results": [],
      "user_id": 15,
      "status": "running",
      "date_started": "2017-11-16 17:46:06.0",
      "num_downloaded": 0,
      "data_type": "property",
      "query": "country:US AND propertyType:\"Single Family Dwelling\"",
      "format": "csv",
      "num_records": 50,
      "total_cost": 50
    }
    ```
  </Step>

  <Step title="Monitor download status" icon="clock" title-type="h3">
    Check on your download status using the "Get Download" call in your Postman collection. Replace `:id` with the `ID` value from the previous step.

    Once completed, the response will look like:

    ```json
    {
      "id": 7,
      "results": [
        "https://datafiniti-downloads.s3.amazonaws.com/15/7_1.csv?..."
      ],
      "status": "completed",
      "num_downloaded": 50,
      "total_cost": 50
    }
    ```

    | Field | Description |
    |-------|-------------|
    | `id` | Unique identifier for the request |
    | `results` | Links to result files (populates as download progresses) |
    | `status` | Download status (`completed` when finished) |
    | `num_downloaded` | Records downloaded so far |
  </Step>

  <Step title="Download result files" icon="file-down" title-type="h3">
    Once `status` shows `completed`, download the data using the URLs in the `results` field.

    <Callout kind="info">
      If you've requested over 10,000 records, you may see multiple result URLs.
    </Callout>

    Copy each URL and paste it into your browser to download the files.
  </Step>

  <Step title="Parse the data" icon="table" title-type="h3">
    Navigate to your download page at [portal.datafiniti.co/download](https://portal.datafiniti.co/download) and open the files.

    <Callout kind="info">
      JSON data is formatted as NDJSON (Newline Delimited JSON) where each line is a separate JSON object. This format handles large datasets better than a single JSON array.
    </Callout>

    Open CSV files in Excel to analyze the data. Use the `province` column to tally properties by state.

    For more use case specific examples, visit [Use Cases for Property Data](/docs/use-cases-for-property-data).
  </Step>
</Steps>
