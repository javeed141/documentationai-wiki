---
title: "Using a Custom Property View On-the-Fly"
description: "Create temporary custom views directly in your Datafiniti API requests."
---

You can create custom views on-the-fly by specifying the exact fields you want in your API request.

## Basic syntax

Specify fields as a comma-separated list in the `view` parameter:

```json
{
  "query": "country:US",
  "num_records": 10,
  "view": "address,city,province,numBedroom,numBathroom"
}
```

## Including nested fields

Access nested field data using dot notation:

```json
{
  "query": "country:US",
  "num_records": 10,
  "view": "address,city,prices.amountMin,prices.dateSeen,brokers.agent"
}
```

## Example: Real estate analysis view

```json
{
  "query": "province:TX AND propertyType:\"Single Family Dwelling\"",
  "num_records": 100,
  "format": "JSON",
  "view": "address,city,postalCode,numBedroom,numBathroom,floorSizeValue,yearBuilt,mostRecentPriceAmount,mostRecentStatus"
}
```

## Example: Broker contact view

```json
{
  "query": "city:Austin AND mostRecentStatus:\"For Sale\"",
  "num_records": 50,
  "format": "CSV",
  "view": "address,city,mostRecentBrokerAgent,mostRecentBrokerCompany,mostRecentBrokerPhones,mostRecentBrokerEmails"
}
```

## Example: Transaction history view

```json
{
  "query": "postalCode:78701",
  "num_records": 25,
  "view": "address,transactions.saleDate,transactions.price,transactions.buyerLastName,transactions.sellerLastName"
}
```

## Tips for custom views

<Callout kind="tip">
  - Start with fewer fields and add more as needed
  - Use `mostRecent*` fields for current values instead of full arrays
  - Test with `num_records: 1` before downloading large datasets
</Callout>

## Limitations

- Field names must match the schema exactly
- Invalid field names are ignored
- Some nested fields may require the parent field to be included
