<Update label="2026-02" description="statusHistory Field & Legacy Deprecation" tags={["schema changes", "deprecation"]}>
  New `statusHistory` field for chronological property status tracking. Legacy `statuses` fields beginning deprecation.

  **New Property Schema Field — `statusHistory`:**
  - Provides a clear, chronological view of a property's status over time (for-sale listings, off-market events)
  - Created because rental statuses were removed from the main statuses array, necessitating a dedicated historical field
  - Each entry includes: `date` (status change date), `dateSeen` (Datafiniti observation date), `sourceURLs`, `type` (sales listing type)
  - Valuable for market trend analyses, property lifecycle models, and investment/off-market intelligence workflows

  **Deprecation Notice — Legacy `statuses` Fields:**
  - Previous `statuses` structure lacked reliable date tracking and historical context
  - Split of rental and sale/off-market statuses left `statuses` field obsolete
  - `statusHistory` is the unified, future-proof replacement
  - Existing fields continue to work for now — strongly recommend updating workflows
  - Ample notice will be provided before any breaking changes

  **Coming This Year:**
  - Improved Autotrace — single API call for property owner contact info
  - AI platform integration — MCP, structured code examples, LLM-friendly tutorials
</Update>

<Update label="2026-03" description="Profile Images, Dev Resources & Deprecation Reminder" tags={["schema changes", "feature", "deprecation"]}>
