# 2017 Data Release Log

<Update label="June 15, 2017" description="Expanded Query & Performance for Business Data" tags={["New Feature", "Breaking Change"]}>

### Expanded Query Capabilities and Accelerated Data Delivery Comes to Business Data

The Elasticsearch migration now extends to Business Data, bringing the same improvements previously launched for Product Data.

**Breaking Changes:**
- A `format` parameter (`JSON` or `CSV`) is now required (defaults to `JSON`).
- Some field names updated per consistent naming conventions. Some fields demoted to `features`.

**Expanded Queries:**
- All fields searchable, including nested fields (except `reviews.text`). Simplified `sourceURLs` querying.
- Comparison operators (`>`, `>=`, `<`, `<=`) supported on numeric fields.

**Performance & Quality:**
- 50x faster ingestion. Full business database downloadable in under 3 hours.
- Improved merging algorithm, comprehensive validation, foreign character support, and consistent `estimated_total` values.
- All fields standardized to camelCase. Multi-valued fields use plural names.

</Update>

<Update label="2017" description="New Product Prices View and CSV Improvements" tags={["New Feature", "Improvement"]}>

### New View for Prices and Changes to CSV Formats

**New `product_pricesFlat` View** — A new view that flattens only pricing information in the product data set, making it easier to collect and analyze individual product prices.

**Updated CSV Format** — Top-level fields (e.g., `name`, `brand`) are now duplicated across all flattened rows for a record. For example, using `products_pricesFlat`, the product name will appear on each price row, simplifying cross-field analysis.

</Update>

<Update label="2017" description="New Developer Documentation for JSON" tags={["Documentation"]}>

### New JSON Code Examples Added

Our [Getting Started with Product Data](https://datafiniti-api.readme.io/docs/getting-started-with-product-data) documentation now includes code examples for working with JSON format data.

**Important note:** Downloaded JSON result files are actually newline-delimited text files containing multiple JSON records, not standard JSON files. The guides include parsing examples to handle this format.

</Update>

<Update label="February 6, 2017" description="Product Data Elasticsearch Migration" tags={["New Feature", "Breaking Change"]}>

### Expanded Query Capabilities and Accelerated Data Delivery

We've completed a full migration of our Product Data to a new Elasticsearch-based backend. [Read the full announcement](https://blog.datafiniti.co/datafiniti-expands-query-capabilities-and-accelerates-data-delivery-performance-with-backend-d675507208e). Business Data and Property Data will follow in the coming months.

**Breaking Changes:**
- A `format` parameter (`JSON` or `CSV`) is now required (defaults to `JSON`).
- Some field names changed per the updated [Product Data Schema](https://datafiniti-api.readme.io/docs/product-data-schema). Some fields demoted to `features`.

**Expanded Queries:**
- All fields are now searchable, including nested fields (e.g., `q=reviews.rating:5`).
- Simplified `sourceURLs` querying — use `q=sourceURLs:amazon` instead of wildcards.
- Comparison operators (`>`, `>=`, `<`, `<=`) supported on numeric fields (e.g., `q=prices.amountMin:>20`).

**Performance & Quality:**
- 50x faster data ingestion. Full product database downloadable in under 3 hours.
- New merging algorithm using multiple keys (`upc`, `manufacturerNumber`, `brand`, etc.) reduces duplicates.
- Comprehensive validation and normalization applied to all historical and new data. Consistent `estimated_total` values.
- All fields standardized to camelCase (e.g., `dateSeen`, `manufacturerNumber`). Multi-valued fields use plural names.

**Usability:**
- New CSV format splits multi-valued fields into individual rows for easier analysis in Excel, R, and Pandas. Legacy format still available.
- More detailed and useful API error messages.

</Update>
