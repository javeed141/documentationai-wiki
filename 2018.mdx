# 2018 Data Release Log

<Update label="2018" description="Major API Overhaul — V4 Launch" tags={["New Feature", "Breaking Change"]}>

### V4 Is Here

We're incredibly excited to announce the arrival of our V4 API with a ton of exciting improvements:

- **Custom Views** — [Create your own views](https://developer.datafiniti.co/v4/reference#create-view) to specify which fields are returned in a search or download, including on-the-fly view creation.
- **Simpler Downloads** — The old `/requests` and `/results` endpoints have been merged into a single [`/downloads`](https://developer.datafiniti.co/v4/reference#get-a-download) endpoint for checking status and retrieving results simultaneously.
- **Fixed Credit Refresh** — The V3 issue of credits not refreshing has been resolved.
- **On-Demand Records** — No more record caps. Use as many records as needed at a slightly higher on-demand price for extras.
- **Better Internal Tools** — Our customer success teams now have improved tools for monitoring API usage and diagnosing issues.
- **Faster Setup** — Instantly download a custom [Postman collection](https://developer.datafiniti.co/v4/reference#postman) for your account.
- **Better Security** — Updated authorization protocol for improved account security.

</Update>

<Update label="2018" description="Expanded Query & Performance for Property Data" tags={["New Feature", "Breaking Change"]}>

### Expanded Query Capabilities and Accelerated Data Delivery Comes to Property Data

We've completed the final phase of our year-long database migration with the launch of new Property Data on the Elasticsearch-based backend.

**Breaking Changes:**
- A `format` parameter (`JSON` or `CSV`) is now required in API calls (defaults to `JSON`).
- Some field names have changed per the updated [Property Data Schema](https://developer.datafiniti.co/docs/property-data-schema). Some fields have been demoted to the `features` field.

**Expanded Queries:**
- All fields are now searchable, including nested fields (e.g., `q=reviews.rating:5`), except `reviews.text`.
- Simplified `sourceURLs` querying — just use `q=sourceURLs:amazon` instead of wildcards.
- Comparison operators (`>`, `>=`, `<`, `<=`) now supported on numeric fields.

**Performance & Quality:**
- 50x faster data ingestion. Full property database downloadable in under 3 hours.
- New merging algorithm reduces duplicates. Comprehensive validation and normalization applied to all data.
- Foreign character support added. Consistent `estimated_total` values across API calls.
- All fields standardized to camelCase (e.g., `dateSeen`, `managedBy`). Multi-valued fields use plural names.

</Update>

<Update label="2018" description="New CSV View for Business Reviews" tags={["New Feature"]}>

### New Business View for Reviews

We've added a new view for business data called [`businesses_all_reviewsFlat`](https://docs.datafiniti.co/docs/businesses_all_reviewsflat), which flattens business reviews when using CSV data for easier analysis.

</Update>

<Update label="2018" description="50 Records Per Preview Call" tags={["Improvement"]}>

### Increasing # of Records Returned by Preview Calls

Preview calls now support up to **50 records** per call, up from the previous limit of 10. Set your `records` parameter to the desired number. Note that requesting more records per call will consume your available downloads faster.

</Update>
