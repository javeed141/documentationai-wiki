## New Fields: `profileImageURLs` & `mostRecentProfileImageURL`

Two new fields in the People Data schema provide access to professional profile images for identity resolution, lead enrichment, and analytics workflows.

- `profileImageURLs` — Professional profile image URL(s) associated with a person
- `mostRecentProfileImageURL` — The most recently observed professional profile image URL
```json title="Example Response"
{
  "people": [
    {
      "firstName": "Jordan",
      "lastName": "Matthews",
      "jobTitle": "Director of Operations",
      "companyName": "Example Logistics Group",
      "profileImageURLs": [
        "https://cdn.example.com/images/jordan-2024.jpg",
        "https://cdn.example.com/images/jordan-2023.jpg"
      ],
      "mostRecentProfileImageURL": "https://cdn.example.com/images/jordan-2024.jpg"
    }
  ]
}
```

For the full schema, see the [People Data Schema](/v3/docs/getting-started-with-people-data-1).

## Deprecation Reminder: Legacy `statuses` Fields

We are reinforcing the deprecation of legacy `statuses` fields in favor of **`statusHistory`**.

- Existing `statuses` fields continue to function for now
- Update your workflows to use `statusHistory`
- Future schema updates will prioritize `statusHistory` as the source of truth

For full details, see the [February 2026 release notes](/v3/changelog/february-2026).
