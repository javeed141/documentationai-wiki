## Webhook Configuration

Set up webhooks to receive real-time events from Dafiniti-Javeed, such as project updates or user actions. Webhooks enable notifications to your external services.

<Steps>
  <Step title="Create Webhook" icon="plus">
    Navigate to your dashboard at `https://dashboard.example.com/settings/webhooks` and click "New Webhook".
  </Step>
  <Step title="Configure Endpoint" icon="link">
    Enter your endpoint URL, such as `https://your-webhook-url.com/webhook`, and select events to subscribe to.
  </Step>
  <Step title="Test and Save" icon="check-circle">
    Send a test payload and verify receipt before saving.
  </Step>
</Steps>

<CodeGroup tabs="JavaScript,Python">
  ```javascript
  // Example webhook payload handler
  app.post('/webhook', (req, res) => {
    const event = req.body.event; // e.g., "project.updated"
    const data = req.body.data;
    console.log(`Received ${event}:`, data);
    res.status(200).send('OK');
  });
  ```
  ```python
  # Example webhook payload handler
  from flask import Flask, request

  app = Flask(__name__)

  @app.route('/webhook', methods=['POST'])
  def webhook():
      event = request.json['event']  # e.g., "project.updated"
      data = request.json['data']
      print(f"Received {event}: {data}")
      return 'OK', 200
  ```
</CodeGroup>

<Callout kind="tip">
  Use HTTPS endpoints for security and include `X-Dafiniti-Signature` header verification with your webhook secret.
</Callout>

## API Endpoints for Custom Integrations

Build custom integrations using our REST API at `https://api.example.com/v1`. Authenticate requests with your API key.

<Request tabs="cURL,JavaScript" show-lines="true">
  ```bash
  curl -X POST https://api.example.com/v1/integrations \
    -H "Authorization: Bearer YOUR_API_KEY" \
    -H "Content-Type: application/json" \
    -d '{"service": "custom", "config": {"url": "https://your-service.com"}}'
  ```
  ```javascript
  const response = await fetch('https://api.example.com/v1/integrations', {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY',
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      service: 'custom',
      config: { url: 'https://your-service.com' }
    })
  });
  ```
</Request>

<Response tabs="200,400">
  ```json
  {
    "id": "int_1234567890abcdef",
    "service": "custom",
    "status": "active",
    "created_at": "2024-01-15T10:30:00Z"
  }
  ```

  ```json
  {
    "error": "invalid_request",
    "message": "Missing required service field"
  }
  ```
</Response>

<ParamField path="service" param-type="string" required="true">
  The integration service name, e.g., `slack` or `custom`.
</ParamField>

<ParamField header="Authorization" param-type="string" required="true">
  Bearer token using `YOUR_API_KEY`.
</ParamField>

## Export and Import Options

Export your data in various formats or import configurations from other tools.

<Tabs>
  <Tab title="CSV Export" icon="download">
    Export projects and users to CSV for analysis.

    ```bash
    curl -H "Authorization: Bearer YOUR_API_KEY" \
      https://api.example.com/v1/export/projects.csv
    ```
  </Tab>
  <Tab title="JSON Import" icon="upload">
    Import settings from JSON files.

    <CodeGroup tabs="cURL,Python">
      ```bash
      curl -X POST https://api.example.com/v1/import \
        -H "Authorization: Bearer YOUR_API_KEY" \
        -F "file=@config.json"
      ```
      ```python
      import requests
      with open('config.json', 'rb') as f:
          response = requests.post(
              'https://api.example.com/v1/import',
              headers={'Authorization': 'Bearer YOUR_API_KEY'},
              files={'file': f}
          )
      ```
    </CodeGroup>
  </Tab>
</Tabs>

<Expandable title="Advanced Export Filters" default-open="false">
  Use query parameters to filter exports:

  | Parameter | Type   | Description                  |
  |-----------|--------|------------------------------|
  | `status`  | string | Filter by `active` or `archived` |
  | `date_from` | date | Start date (ISO format)     |
  | `limit`   | number | Max records (default: 1000) |
</Expandable>
