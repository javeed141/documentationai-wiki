## v1.2.0

## Upgrading to the Latest Version

Follow these steps to upgrade Dafiniti-Javeed safely.

<Steps>
  <Step title="Backup Configuration" icon="database">

    Export your current settings:

````bash
dafiniti-javeed config export > backup.json
````

  </Step>
  <Step title="Update Package" icon="download">

    <CodeGroup tabs="npm,yarn">
````bash
npm install dafiniti-javeed@latest
````
````bash
yarn add dafiniti-javeed@latest
````
    </CodeGroup>

  </Step>
  <Step title="Run Migrations" icon="settings">

    Apply database changes:

````bash
dafiniti-javeed migrate up
````

  </Step>
  <Step title="Restart Service" icon="refresh-cw">

    Restart your instance and verify at `https://dashboard.example.com`.

  </Step>
</Steps>

<Callout kind="alert">
  Always test upgrades in a staging environment first. Review migration notes for each version.
</Callout>

## Migration Notes

<Expandable title="Detailed v1.2.0 Migration Guide" default-open="true">

If upgrading from v1.1.0, note these changes:

| Change | Action Required | Impact |
|--------|-----------------|--------|
| New `webhook_url` param | Update config with your endpoint | Optional feature |
| Removed legacy `api_v0` endpoints | Migrate to `/v1` paths | Breaking for old integrations |
| `Authorization` header now required | Add `Bearer YOUR_TOKEN` | Security enhancement |

For full details, check the [upgrade script](https://github.com/dafiniti-javeed/upgrade-v1.2.0.sh).

</Expandable>

<Expandable title="Deprecated Features">

- `legacy-auth` mode will be removed in v2.0.0. Switch to `mfa` now.
- Old dashboard URL `https://dashboard.example.com/v0` redirects but logs warnings.

</Expandable>
